# セキュリティ設定解説 (next.config.ts)

本書では、アプリケーション（Pancolle）のセキュリティを強固にするために `next.config.ts` に導入された各セキュリティ設定項目とその目的について解説します。

---

## 1. 基本設定

### poweredByHeader: false
* **概要**: レスポンスヘッダーに含まれる `X-Powered-By: Next.js` を非表示にします。
* **目的**: 攻撃者に対してシステムが Next.js で構築されているという情報を隠蔽（フィンガープリントの防止）し、Next.js 固有の既知の脆弱性を狙った攻撃のリスクを低減します。

---

## 2. HTTP セキュリティヘッダー (Security Headers)

ブラウザのセキュリティ機能を強制・有効化するためのヘッダー群です。

### X-Frame-Options: DENY
* **概要**: 他のサイトの `<iframe>` などの要素内に、自サイトが表示されることを全面的に禁止します。
* **目的**: **クリックジャッキング攻撃**（透明な iframe を重ねてユーザーに意図しない操作を行わせる攻撃）を防止します。

### X-Content-Type-Options: nosniff
* **概要**: ブラウザによる MIME タイプの自動スニッフィング（コンテンツの内容からファイル種類を勝手に判断する機能）を無効化します。
* **目的**: サーバーが返した `Content-Type` ヘッダーをブラウザに強制追従させ、テキストファイルや画像ファイルに見せかけた悪意あるスクリプト（XSSペイロード）が実行されるのを防ぎます。

### Referrer-Policy: origin-when-cross-origin
* **概要**: 同一オリジ内での遷移ではフルURLを、外部サイトへの遷移（クロスオリジン）ではオリジン（ドメイン名）のみを `Referer` ヘッダーとして送信します。
* **目的**: 外部サイトへリンクで遷移する際、URLのクエリパラメータなどに含まれる可能性のある機密情報（セッションIDやトークンなど）が外部に漏洩するのを防止します。

### Strict-Transport-Security (HSTS)
* **設定値**: `max-age=31536000; includeSubDomains; preload`
* **概要**: ブラウザに対して、今後1年間（31536000秒）は常に HTTP ではなく HTTPS を使用して通信することを強制します。
* **目的**: **中間者攻撃 (MITM)** や、HTTP通信を強制的に傍受する SSL Strip 攻撃を防御します。

### Permissions-Policy
* **設定値**: `camera=(), microphone=(), geolocation=()`
* **概要**: ブラウザの特定のハードウェア/デバイス機能へのアクセスを制限します。
* **目的**: 本アプリではカメラやマイク、位置情報を直接ブラウザAPIから利用しないため、明示的に拒否（空のリスト `()`）に設定しています。これにより、万が一XSS等で不正なスクリプトが挿入された場合でも、ユーザーのデバイス機能が盗聴などに悪用されるリスクを遮断します。

## 3. コンテンツセキュリティポリシー (Content Security Policy: CSP)

信頼できるリソース（スクリプト、スタイル、画像など）の読み込み元を厳格に制限する、XSS対策において最も強力な設定です。

### default-src 'self';
* **意味**: すべてのリソースのデフォルトの許可元を「同一オリジン（自サイト）」に制限します。明示的に他で上書きしない限り、外部リソースは一切遮断されます。

### script-src 'self' 'unsafe-eval' 'unsafe-inline';
* **意味**: スクリプトの実行を自サイト内、および一部のインラインスクリプトや `eval`（Next.js の開発・ハイドレーション動作に必要な範囲）に制限します。
* **注意**: 本番環境ではさらに厳格化（`'unsafe-inline'` の排除と Nonce の導入）を検討することが望ましいですが、Next.jsの標準的な静的/動的レンダリングを阻害しないための現実的な初期設定です。

### style-src 'self' 'unsafe-inline';
* **意味**: スタイルの読み込みを自サイトとインラインスタイルに制限します。Tailwind CSS などが動的にスタイルを適用するために `'unsafe-inline'` が必要となります。

### img-src 'self' data: blob: https://lh3.googleusercontent.com;
* **意味**: 画像の読み込み元を制限します。
  * `'self'`: 自サイト内の画像。
  * `data:`: インラインのベース64画像。
  * `blob:`: **【重要】** `UploadPanel.tsx` などで納品書画像をアップロードした際、画面上でプレビュー表示するために使用する `URL.createObjectURL`（`blob:` URL）を許可します。
  * `https://lh3.googleusercontent.com`: **【認証連携用】** Googleログイン（NextAuth.js）経由で取得したユーザーのプロフィール画像を表示できるようにするために許可しています。

### font-src 'self' data:;
* **意味**: フォントファイルを自サイトおよび data URI からのみ読み込むことを許可します。

### object-src 'none';
* **意味**: `<object>`、`<embed>`、`<applet>` タグによるプラグイン（FlashやJavaアプレットなど）の読み込みを全面的に禁止します。悪意あるバイナリ実行を防ぎます。

### frame-ancestors 'none';
* **意味**: 他のサイトがこのアプリを `<iframe>` 等に埋め込むことを禁止します（`X-Frame-Options: DENY` の現代版ブラウザ向け設定）。

### upgrade-insecure-requests;
* **意味**: ページ内に HTTP（暗号化なし）のリンクやリソースが含まれていた場合、ブラウザ側で自動的に HTTPS へアップグレードしてリクエストさせます。

## 4. Next.js 固有の外部リソース制限

### images.remotePatterns
* **設定**: `hostname: 'lh3.googleusercontent.com'`
* **概要**: Next.js の `<Image />` コンポーネントが最適化・配信を許可する外部ドメインを限定します。
* **目的**: オープンリダイレクタや画像プロキシの悪用を防ぎ、アプリケーションが信頼するGoogleのユーザーアイコン配信サーバーからのみ安全に画像をロードします。
